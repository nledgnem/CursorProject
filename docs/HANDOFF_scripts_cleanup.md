# Handoff: Scripts Folder Cleanup

**Context doc to load first:** `docs/DATA_LAKE_CONTEXT.md` and `.cursorrules`.

## Goal

Reduce `scripts/` from 136 files to a clean, maintainable set. Separate production code from dead code and one-shot diagnostics, without breaking the production pipeline or alert runners.

## Non-negotiable constraints

- **Work on a branch.** `claude/scripts-cleanup-<date>` or similar. Never delete directly on main.
- **Commit in tiered chunks** (Tier 1, Tier 2, Tier 3 below) so each can be reverted independently if something breaks.
- **Verify nothing breaks between tiers.** After each tier's commits, trigger `python run_live_pipeline.py --skip-ingestion` locally (or on Render) to smoke-test that imports and script resolutions still work. If any tier's commits break imports, STOP and revert that tier before proceeding.
- **Do not touch** any script referenced by:
  - `run_live_pipeline.py` (pipeline orchestrator at repo root)
  - `start_render.sh` (Render boot script at repo root)
  - `scripts/apathy_alert_runner.py`, `scripts/danlongshort_alert_runner.py`, `scripts/danlongshort_bot.py` (background runners)
  - `system_heartbeat.py` at repo root (daemon)
- **Ask before deleting anything you're <90% confident is orphaned.** Moving to an `archive/` subdirectory is always safer than `git rm`.

## Final state goal

- `scripts/` contains only scripts that:
  1. Are called by the live pipeline, or
  2. Are called by alert runners / bots / heartbeats, or
  3. Are operator tools an engineer would run manually (apathy_*, xref_*, check_lake, etc.)
- `notebooks/` contains all diagnostic + EDA scripts
- `scripts/archive/` contains orphans with a README explaining why each was archived
- `.png` files generated by diagnostics are gitignored and don't pollute the tree

## When done

1. Push the branch.
2. Update `ARCHITECTURE.md` (§"The Lab" section) if the directory conventions have shifted enough to warrant it.
3. Add a note to `docs/DATA_LAKE_CONTEXT.md` §13 (known gaps / current work) marking the cleanup as done, with date.
4. Summarize: "Reduced scripts/ from 136 to N files. Archived X, moved Y to notebooks, deleted Z PNG outputs."

## Things NOT in scope

- Refactoring any surviving script. Cleanup is about separation, not rewriting.
- Touching `src/`, `configs/`, `data/`, `dashboards/`. Out of scope.
- Deleting anything from git history. `git rm` + commit is enough; we don't need to rewrite history.
